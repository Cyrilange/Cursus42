# 42 Cursus — Low-Level Programming & Systems Development

This repository contains my work throughout the **42 School Cursus**, an intensive, project-based software engineering program focused on **low-level programming**, **systems development**, and **algorithmic problem solving**.

Projects are developed primarily in **C and C++**, under strict constraints:
- No external libraries unless explicitly allowed
- Manual memory management
- Strong focus on performance, correctness, and robustness
- Compliance with the 42 coding standard (Norm)

---

## 🧩 Projects Overview

### 📦 libft
Reimplementation of core C standard library functions.

**Concepts:**
- Memory management
- Pointers and arrays
- String manipulation
- Linked lists

This project builds the foundation for all subsequent C projects.

---

### 🖨️ ft_printf
Custom implementation of the `printf` function.

**Concepts:**
- Variadic functions
- Format parsing
- Buffering and output optimization
- Strict behavior matching libc

---

### 📄 get_next_line
Function that reads a file descriptor line by line.

**Concepts:**
- Static variables
- File descriptors
- Buffer management
- Memory safety

---

### 📡 Minitalk
Client-server communication using UNIX signals.

**Concepts:**
- Inter-process communication (IPC)
- Signal handling (`SIGUSR1`, `SIGUSR2`)
- Bitwise operations
- Synchronization

---

### 🔢 push_swap
Sorting algorithm project with strict operation constraints.

**Concepts:**
- Stack-based sorting
- Algorithmic optimization
- Complexity analysis
- Instruction minimization

---

### 🎮 so_long
2D game using a lightweight graphics library.

**Concepts:**
- Event handling
- Game loops
- Map parsing and validation
- Basic graphics rendering

---

### 🎮 Cub3D
3D raycasting engine inspired by early FPS games.

**Concepts:**
- Raycasting and 3D projection
- Mathematical computations
- Texture mapping
- Player movement and collision detection
- Real-time rendering

---

### 🧠 minishell
Reimplementation of a minimal UNIX shell.

**Concepts:**
- Command parsing and execution
- Process creation (`fork`, `execve`)
- Pipes and redirections
- Environment variables
- Signal handling
- File descriptors

One of the most system-intensive projects of the cursus.

---

### 🧵 philo (Philosophers)
Concurrency and synchronization project.

**Concepts:**
- Multithreading
- Mutexes
- Deadlock prevention
- Time management
- Race condition handling

---

### 🧠 C++ Modules
Introduction to **modern C++** and Object-Oriented Programming.

**Concepts:**
- Classes and objects
- Inheritance and polymorphism
- Encapsulation
- Abstract classes and interfaces
- STL containers and algorithms
- Exception handling

Marks the transition from procedural C to structured C++ design.

---
